## 5. ­¤öŹ Backend Deep Dive: QR Handshake

This is the core trust mechanism of utdash.

### Flow

1. **Claim**
   - Provider claims order
   - Server generates:
     ```
     secret_token = "abc-123"
     ```

2. **Display**
   - Token sent to Provider
   - Rendered as QR code

3. **Scan**
   - Requester scans QR
   - Sends:
     ```
     POST /verify { token: "abc-123" }
     ```

4. **Validation**
    ```sql
    UPDATE orders 
    SET status = 'DELIVERED' 
    WHERE handshake_token = $1 
      AND status = 'CLAIMED';
    ```

5. **Payout**
    - If successful: Move funds from Escrow ŌåÆ Provider Available Balance
